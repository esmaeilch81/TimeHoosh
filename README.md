# سامانه ثبت ساعت کاری پرسنل (شیفتی)

برنامه‌ای تک‌فایلی به زبان Go با رابط کاربری وب (راست‌چین، فونت وزیرمتن)، تاریخ شمسی، ثبت ورود/خروج زنده، ثبت دستی، و خروجی اکسل حرفه‌ای با نمودار.

## امکانات
- مدیریت پرسنل: افزودن / غیرفعال کردن بدون پاک شدن سابقه
- ثبت ورود و خروج با یک دکمه، تاریخ و ساعت به‌صورت شمسی
- تشخیص خودکار شیفت‌های چندروزه (شب‌کار، ۲۴ ساعته) چون کل لحظه‌ی ورود و خروج ذخیره می‌شود، نه فقط ساعت
- ثبت دستی برای زمانی که فرد یادش رفته ثبت کند، همراه با برچسب «دستی» و فیلد توضیح
- هشدار شیفت‌های باز در داشبورد
- خلاصه ساعت کارکرد ماهانه هر نفر در داشبورد
- خروجی اکسل با ستون‌های خواسته‌شده + شیت خلاصه هر نفر + نمودار ستونی، برای این ماه / ماه قبل / بازه دلخواه / کل تاریخچه
- بک‌آپ خودکار از داده‌ها بعد از هر تغییر (پوشه `data/backups`)
- جلوگیری از ثبت شیفت‌های کمتر از یک دقیقه (چه ثبت زنده، چه دستی)
- جلوگیری از تداخل زمانی: اگر برای یک نفر هم ثبت زنده و هم ثبت دستی در یک بازه‌ی هم‌پوشان بخواهید بزنید، برنامه خطا می‌دهد و اجازه نمی‌دهد
- صفحه‌بندی در بخش گزارش‌ها وقتی تعداد رکوردها زیاد باشد
- نمایش درست و مرتب روی گوشی موبایل

تمام داده‌ها در یک فایل دیتابیس SQLite به نام `data/data.db` کنار فایل اجرایی ذخیره می‌شود؛ برای این تعداد نفر (۴-۵ نفر) کاملاً کافی و سریع است و نیازی به نصب دیتابیس جداگانه (مثل SQL Server) نیست — SQLite در خود فایل exe جای گرفته و نصب جدا نمی‌خواهد.

---

## مرحله ۱: نصب Go (فقط یک‌بار)

اگه Go روی سیستم‌تون نصب نیست:
1. به آدرس https://go.dev/dl/ بروید و نصب‌کننده ویندوز (`go1.xx.windows-amd64.msi`) را دانلود و نصب کنید.
2. بعد از نصب، یک CMD یا PowerShell جدید باز کنید و تست کنید:
   ```
   go version
   ```
   باید نسخه‌ی Go را نشان بدهد.

> نکته: با توجه به این‌که قبلاً برای نصب `gopls` با محدودیت شبکه مواجه شده بودید، ممکنه لازم باشه اینجا هم پراکسی یا یک GOPROXY جایگزین تنظیم کنید. اگه دستور `go mod tidy` یا `go build` در مرحله بعد با خطای شبکه متوقف شد، این را در همون ترمینال اجرا کنید و دوباره تلاش کنید:
> ```
> go env -w GOPROXY=https://goproxy.io,direct
> go env -w GOSUMDB=off
> ```

## مرحله ۲: دریافت وابستگی‌ها

فایل فشرده پروژه را باز کنید (پوشه `shift-tracker`)، وارد آن شوید و در ترمینال/CMD همان‌جا:

```
go get modernc.org/sqlite@latest
go mod tidy
```

این دستور کتابخانه‌های لازم (تبدیل تاریخ شمسی، ساخت اکسل، و درایور SQLite) را دانلود می‌کند. درایور SQLite استفاده‌شده (`modernc.org/sqlite`) کاملاً به زبان Go نوشته شده و به CGO/gcc نیازی ندارد، پس دستور معمولی `go build` روی ویندوز بدون هیچ ابزار اضافه‌ای کار می‌کند.

⚠️ چون در محیط من (سندباکس) دسترسی شبکه نداشتم، نتوانستم خودم `go mod tidy` را اجرا کنم و `go.sum` را کامل کنم؛ حتماً همین یک‌بار این دو دستور بالا را اجرا کنید، وگرنه `go build` با خطای «missing go.sum entry» متوقف می‌شود.

## مرحله ۳: ساخت فایل اجرایی (exe)

```
go build -o shift-tracker.exe .
```

بعد از چند ثانیه یک فایل `shift-tracker.exe` کنار پوشه‌ی `web` ساخته می‌شود.

⚠️ نکته مهم: پوشه‌ی `web` (که شامل صفحه و استایل و جاوااسکریپت است) داخل خودِ فایل exe جای‌گذاری (embed) می‌شود، پس بعد از ساخت exe فقط کافیه همین یک فایل exe را جایی که می‌خواهید کپی کنید — نیازی به بردن پوشه `web` نیست. فقط پوشه‌ی `data` (که خودش خودکار ساخته می‌شود) کنار exe باقی می‌ماند.

## مرحله ۴: اجرا

روی `shift-tracker.exe` دوبار کلیک کنید یا در ترمینال بزنید:
```
shift-tracker.exe
```
پیامی شبیه این می‌بینید:
```
======================================
 سامانه ثبت ساعت کاری پرسنل
 آدرس: http://localhost:8080
 برای خروج: Ctrl+C
======================================
```
حالا مرورگر را باز کنید و به آدرس زیر بروید:
```
http://localhost:8080
```

برای این‌که هر کدام از همکارهاتون از کامپیوترهای دیگه هم بتونن وصل شن (همون شبکه محلی)، آدرس IP کامپیوتری که سرور روش اجراست رو پیدا کنید (با دستور `ipconfig`) و به‌جای `localhost` از همون IP استفاده کنید، مثلاً `http://192.168.1.10:8080`.

## نگه‌داشتن برنامه همیشه روشن

اگه می‌خواهید هر بار که ویندوز روشن می‌شه این برنامه هم خودکار اجرا بشه، یک shortcut از `shift-tracker.exe` بسازید و در پوشه Startup ویندوز (مسیر `shell:startup` در Run) قرار بدید.

## پشتیبان‌گیری

هر بار که چیزی تغییر کنه (ورود، خروج، ثبت دستی، ...) یک نسخه‌ی پشتیبان کامل از دیتابیس با مهر زمانی در `data/backups/` ذخیره می‌شه (تا ۶۰ نسخه‌ی آخر نگه داشته می‌شه). برای بازگردانی، کافیه فایل بک‌آپ دلخواه (یک فایل `.db`) رو کپی کنید و به‌جای `data/data.db` بگذارید (وقتی برنامه خاموش است).

## قوانین ثبت شیفت

- شیفتی که کمتر از یک دقیقه طول بکشد (چه با دکمه‌ی ورود/خروج زنده، چه با ثبت دستی) ذخیره نمی‌شود؛ برنامه خطای مناسب نشان می‌دهد.
- برای هر نفر، بازه‌های زمانی شیفت‌ها (زنده یا دستی) نمی‌توانند با هم تداخل داشته باشند. مثلاً اگر یک نفر همین الان با دکمه‌ی زنده شیفتش باز باشد، نمی‌شود برایش یک شیفت دستی هم‌پوشان در همان بازه ثبت کرد؛ برنامه با پیام خطا مشخص می‌کند دقیقاً با کدام شیفت دیگر تداخل دارد. اگر بازه‌ها با هم تداخل نداشته باشند (مثلاً صبح دستی، عصر زنده)، مشکلی نیست و هر دو ثبت می‌شوند.

```

├── internal/

│   ├── store/                  ذخیره‌سازی SQLite + بک‌آپ
│   ├── jalali/                 تبدیل و فرمت تاریخ شمسی
│   └── excelexport/            ساخت فایل خروجی اکسل
└── web/static/                 index.html + style.css + app.js (RTL + وزیرمتن)
```
